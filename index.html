文件上传漏洞学习

漏洞说明

文件上传漏洞通常发生在应用程序允许用户上传文件但未正确验证这些文件时。攻击者可以上传恶意文件（如Web shell）从而危害服务器。

常见漏洞类型：

  • insufficient_type_validation - 仅依赖客户端验证
  • insufficient_file_extension_validation - 黑名单方法不完善
  • unrestricted_file_upload - 无任何限制

防护措施

要防止文件上传漏洞，应采取多层防御策略：

1. 文件类型验证

// 服务器端MIME类型检查
$allowed = ['image/jpeg', 'image/png', 'text/plain'];
if (!in_array($_FILES['file']['type'], $allowed)) {
  die('Invalid file type');
}

2. 文件扩展名验证

// 白名单方法更安全
$allowed_ext = ['jpg', 'png', 'txt'];
$ext = strtolower(pathinfo($name, PATHINFO_EXTENSION));
if (!in_array($ext, $allowed_ext)) {
  die('Invalid file extension');
}

3. 其他防护措施

  • 将上传的文件存储在Web根目录之外
  • 重命名上传的文件
  • 使用病毒扫描程序检查上传的文件
  • 设置适当的文件权限
